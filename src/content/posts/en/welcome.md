## What You'll Find Here

This blog documents security research, red team techniques, CTF solutions, and the occasional deep-dive into a CVE or exploit chain.

Posts are written with defenders in mind too — understanding how an attack works is half the job; the other half is knowing what artefacts it leaves behind.

## Categories

| Category | What It Covers |
|---|---|
| **Web Exploitation** | SQLi, XSS, SSRF, deserialization, logic flaws |
| **Active Directory** | Kerberoasting, ACL abuse, lateral movement |
| **Red Team Ops** | C2 frameworks, evasion, phishing infrastructure |
| **CTF Write-ups** | Step-by-step solutions from competitions |
| **Tooling** | Custom scripts and tool releases |
| **Malware Analysis** | Static and dynamic reversing |

## Lab Setup

Most techniques are demonstrated in purpose-built home lab environments:
- Windows Server AD domain (various patch levels)
- Vulnerable-by-design web apps (DVWA, HackTheBox, TryHackMe)
- AWS/Azure sandboxed tenants for cloud research

## Responsible Disclosure

CVEs and vulnerabilities are disclosed to vendors before publishing. Write-ups go live after the patch window closes or a CVE is assigned.
